Эта страница связывает всё, что нужно для первого запуска и демонстрации проекта: demo-конфиги, основной pipeline, approval/HITL интерфейсы, README и итоговые HTML-артефакты. Сначала запускайте offline demo, затем переходите к dashboard, source approval и review workspace.

text-first offline-first baseline online = extension HITL required

С чего начать

Для первого запуска не трогайте online-конфиги. Безопасный путь: `demo_fitness`, затем `run_dashboard.html`, потом `source_approval_workspace.html` и `review_workspace.html`.

  1. Откройте VS Code task `run demo_fitness` или скопируйте команду ниже.
  2. После завершения откройте dashboard и EDA.
  3. Если нужен manual approval источников, перейдите в source approval workspace.
  4. Если нужна ручная правка low-confidence разметки, перейдите в review workspace.

Что важно помнить

Launcher не запускает Python сам по себе: он служит понятной точкой входа в проект. Запуск делайте через VS Code tasks или команду в PowerShell.

После каждого run основная точка входа в результаты: ../reports/run_dashboard.html.

Команда для первого запуска

Этот сценарий безопасен, воспроизводим и не зависит от внешней сети.

.\.venv\Scripts\python.exe run_pipeline.py --config configs\demo_fitness.yaml

Команда для source approval

После запуска можно сразу открыть approval workspace для ручного выбора разрешённых источников.

start .\reports\source_approval_workspace.html

Gemini API key

LLM нужен только на этапе auto-labeling. Без ключа проект остаётся рабочим и переключается на mock/offline-safe path.

$env:GEMINI_API_KEY="paste-your-key-here"
Не сохраняйте настоящий ключ в HTML или в коммит. Лучше задавать его в терминале перед online/LLM-запуском.

Рекомендуемые VS Code tasks

  • `open launcher`
  • `run demo_fitness`
  • `run demo_fitness and open dashboard`
  • `open dashboard`
  • `open source approval workspace`
  • `open review workspace`

Online и approval path

Online-режим сейчас полезен как расширение: shortlist, API/scrape/Hugging Face collection и ручное source approval. Начинать знакомство с проектом всё равно лучше с offline demo, а потом уже переходить к `text_topic_template.yaml` и `source_approval_workspace.html`.
